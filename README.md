# 三鍋卓也 ポートフォリオサイト

ITコンサルタント・エンジニアとしての経歴・スキル・実績をまとめた個人ポートフォリオサイトです。

**公開URL**: https://minabetakuya.github.io/minabe_Portfolio/

---

## アーキテクチャ選定理由

### ビルドツールなしの静的HTML構成を採用した理由

本サイトはReactやNext.jsなどのフレームワークを使わず、単一のHTMLファイル + CSS + JSというシンプルな構成を採用しています。

- **ポートフォリオサイトの要件にフレームワークは過剰**: 本サイトは単一ページの情報掲載サイトであり、ルーティング・状態管理・API通信といったフレームワークが解決する課題がそもそも存在しない。必要十分な技術選定を優先した
- **ビルド工程ゼロによるデプロイの簡素化**: `npm build`やCI/CDパイプラインが不要。HTMLファイルをリポジトリにプッシュするだけでデプロイが完了するため、運用コストを最小限に抑えられる
- **依存パッケージの排除による長期的な安定性**: `node_modules`に依存しないため、パッケージの破壊的更新やセキュリティ脆弱性の影響を受けない。数年後にメンテナンスする際もそのまま動作する

### CDN経由のライブラリ活用

ビルドツールを持たない構成でも、CDN経由でモダンなライブラリを活用しています。

| ライブラリ | 役割 | 採用理由 |
|---|---|---|
| **TailwindCSS (CDN)** | ユーティリティCSS | クラスベースで迅速にレスポンシブデザインを実装。CDN版でビルド不要 |
| **Alpine.js** | 軽量リアクティブUI | ハンバーガーメニューなど小規模なインタラクションに最適。Vue的な記法をHTML属性だけで実現 |
| **AOS** | スクロールアニメーション | data属性を付与するだけでフェードインなどのアニメーションを実装可能 |
| **Font Awesome** | アイコンフォント | SNSアイコンなどを手軽に表示。SVG管理の手間を削減 |
| **Google Fonts** | Webフォント | Noto Sans JP / Noto Serif JP で日本語の可読性とデザイン性を両立 |

## ディレクトリ構成

```
minabe_Portfolio/
├── index.html              # メインページ（全セクションを単一ファイルに集約）
├── 404.html                # GitHub Pages用カスタム404ページ
├── robots.txt              # 検索エンジンクローラー制御
├── sitemap.xml             # サイトマップ（SEO対策）
├── favicon.ico             # ファビコン
├── apple-touch-icon.png    # iOS ホーム画面用アイコン
├── icon.png / icon.svg     # 汎用アイコン
├── assets/
│   ├── css/
│   │   └── style.css       # カスタムCSS（CSS変数によるデザイントークン管理）
│   ├── js/
│   │   └── main.js         # AOS初期化・ナビゲーション制御
│   └── images/             # プロフィール画像・OGP画像・QRコード等
├── LICENSE.txt
└── README.md
```

### 構成の意図

- **単一HTMLファイル（SPA的構成）**: セクション（Hero / About / Skills / Works / Career / Contact）を1ファイルに集約。ページ遷移が発生しないため表示速度が速く、アンカーリンクによるスムーズスクロールでUXを確保している
- **assets/ 配下のCSS/JS/画像の分離**: HTMLからスタイル・スクリプト・画像を分離することで、関心の分離を維持。ファイル単体の見通しを良くし、変更時の影響範囲を限定している
- **CSS変数（デザイントークン）による色管理**: `style.css`内の`:root`変数を変更するだけで、サイト全体の配色を一括変更できる設計。テーマカラーの変更が1箇所で完結する
- **ルート直下のSEO関連ファイル**: `robots.txt`・`sitemap.xml`・ファビコン類はGitHub Pagesのルート直下に配置する必要があるため、この構成としている

## 使用言語の意図

### HTML

サイトの構造定義に使用。セマンティックなセクション分割（`<section>`, `<nav>`, `<footer>`）によりアクセシビリティとSEOに配慮している。OGP・Twitter Cardのメタタグも記述し、SNSシェア時の表示を最適化している。

### CSS（カスタムCSS + TailwindCSS）

- **TailwindCSS**: レイアウト・レスポンシブ・余白などの汎用的なスタイリングはTailwindのユーティリティクラスで対応。HTMLを見るだけでデザインが把握できるため、保守性が高い
- **カスタムCSS（style.css）**: Tailwindでは表現しきれない独自デザイン（グラデーション背景・カードスタイル・アニメーション・ホバーエフェクト等）をカスタムCSSで補完。CSS変数でデザイントークンを管理し、色やフォントの一括変更を可能にしている

### JavaScript

- **Alpine.js**: ハンバーガーメニューの開閉やリサイズ時の自動クローズなど、軽量なUI操作をHTMLの`x-data`属性で宣言的に記述。Reactのような仮想DOMを必要としない小規模インタラクションに最適
- **Vanilla JS（main.js）**: AOS初期化とナビゲーションのスクロール検知のみ。ライブラリに依存しない最小限のスクリプトで、スクロール時のナビバーシャドウ表示を制御している

## ホスティング選定の意図

### GitHub Pagesを採用した理由

| 観点 | GitHub Pages | 他の選択肢との比較 |
|---|---|---|
| **コスト** | 完全無料 | Vercel/Netlifyも無料枠があるが、静的HTMLにはGitHub Pagesで十分 |
| **デプロイフロー** | `git push`のみ | ビルドコマンドやCI設定が不要。リポジトリ更新＝デプロイ完了 |
| **HTTPS** | 標準で提供 | 追加設定なしでSSL証明書が自動適用される |
| **可用性** | GitHubのインフラに依拠 | 高い稼働率。ポートフォリオ用途には十分な信頼性 |
| **ソースコード管理との一体化** | リポジトリ＝公開サイト | コードの変更履歴とデプロイ履歴が一致し、運用がシンプル |

- **サーバーサイド処理が不要**: 本サイトはフォーム送信をGoogle Forms外部リンクで処理しており、バックエンド機能を一切必要としない。そのため静的ホスティングで完結する
- **独自ドメイン不要**: ポートフォリオとしての用途では`github.io`ドメインで十分であり、ドメイン管理コストを削減している
- **技術スタックとの親和性**: ビルド工程のない静的HTML構成と、ビルド不要でデプロイできるGitHub Pagesは最適な組み合わせ

## セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/minabetakuya/minabe_Portfolio.git

# ローカルで確認（任意のローカルサーバーで起動）
# 例: VS Code の Live Server 拡張、または Python の簡易サーバー
python3 -m http.server 8000
```

ビルド手順は不要です。クローンしたファイルをそのままブラウザで開くか、ローカルサーバーで配信するだけで動作します。
